Security is imperative to our usage of the web every day, and through encryption of various types we can confidently browse the web and use services without worrying about secure information getting released such as passwords, credit card details, private messaging and email, as well as much much more

Public-Key Cryptography RSA - Rivest Shamir Adleman

Hide Shared Information From Interceptors

Public-Key Cryptography is a type of cryptography that allows two users to send information from one to each other in a way that cannot be broken if a malicious user intercepts the message while in transit, it does this by using multiple components, such as public and private keys, it is highly secure and is commonly used for applications such as sharing vital small amounts of information like sensitive messages and encryption keys, examples of this are the RSA and ECC algorithms

Symmetric Encryption AES - Advanced Encryption Standard

Securely and Quickly Encrypt Large Data

Symmetric encryption is a type of encryption which uses a single key to securely encode and decode information, it is very useful for encrypting large amounts of data quickly and efficiently, because of this it is commonly used for protecting files like documents, photos, emails, applications, and more, symmetric encryption is the most common type of encryption people are familiar with, examples of this are the AES and DES algorithms

Digital Signatures DSA - Digital Signature Algorithm

Prove Authenticity of Data and Messages

Digital signatures are important to secure data transfer accross the web, it provides a way to verify the authenticity of messages being sent accross the internet, which is necessary to prevent forging of requests and verifying the identity of hosts, it is used in most every portion of the web incuding email, messaging, web requests, and more, examples of this are the DSA and RSA algorithms

Hashing Algorithms SHA2 - Secure Hash Algorithm

Irreversable Encoding of Information

Hashing is the best way to make a one way conversion from data to a string of seemingly random bytes, it is not reversable but always provides the same output from the same input this is important to the function of computers ans the web as a whole for two main reasons, it allows sensitive information like passwords to be stored as a hash so if they get breached they cannot be used, and also to verify files or messages have not been changed through checksums, examples of this are the SHA2 and MD5 algorithms
