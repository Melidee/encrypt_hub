SHA2: Securely Hash Information

What is SHA2 actually

SHA2, the Secure Hash Algorithm 2, is an algorithm made to create 256 bit hashes from a given input, a hash is a seemingly random string of bits that has some very specific properties, they can be generated from a file or other string of bytes that is any size, and always produces the same output given the same input, and is incredibly difficult to reverse the input from a given hash

How is SHA2 used

SHA2 is used in a variety of cryptographic use cases, for example it can be used to hide passwords so they arent stored in plaintext, it can be used to verify a file has not been changed or tampered with, and it can be used as a part of another algorithm, for example the DSA algorithm uses it when signing documents

How does it hide passwords

Passwords are incredibly important across computing and the internet. SHA2 is the most modern hash used to hide passwords and protect them in the case of a breach, it works like this, a user enters their password when creating their account, then the password is hashed before being put into a database, when the user goes to login the password is first hashed, then that hash is compared to the database to check if they match, in the case of a database breach, the attacker would not have direct access to any of the passwords, just the hashes, which arent useful on their own

How does it verify documents

When a sensitive file or document is being sent across an insecure medium like the internet, it is important to know that the file is not being tampered with by a third party, hashes help with this issue by allowing you to verify the document by hashing it, suppose you wanted to secretly send the word "color" across an insecure path, you could create a hash in an algorithm called MD5 which results in "70dda5dfb8053dc6d1c492574bce9bfd" as an output, but if the string was tampered with to "colour" you would get a hash like "296cc9a02b167493f6ff885c48ca71ff", the host recieving this message could hash it and realize that the hash is different than the expected one, and determine that it must have been changed
