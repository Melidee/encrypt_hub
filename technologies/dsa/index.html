DSA: Digitally Sign Data

What is DSA actually

DSA is an algorithm which is used to sign documents and data before they are sent as a message to provide authenticity and integrity to the message, authenticity of a message is knowing the true identity of the sender, this prevents anyone from being able to pretend to be someone they arent, integrity is knowing that the message was not tampered with, corrupted, or otherwise altered during it's transit, which is important to be sure you are not recieving fake or forged data

How is DSA used

DSA is used all across the web whenever there needs to be absolute certainty that the message being sent has both authenticity and integrity, a critical example of this would be a banking service, the bank would need to know that a request is authentic, not coming from someone pretending to be a valid account holder, and that it has integrity, so that no malicious other party could change the message to do something like change who money is being sent to or how much is being sent

How is it different than Asymmetric Encryption

CFB) Cipher FeedBack is an AES mode which does not need padding, and uses ciphertext feedback to encode other blocks, it is generally secure but can be insecure to forging and bad blocks corrupt all future blocks
OFB) output feedback creates a stream independent of the plaintext, it needs to create an independent nonce or IV for each message. It is generally secure and bad bits dont affect other bits.
CTR) Counter mode works by encoding the blocks using a count and it is the fastest mode, it is not affected downstream by broken blocks, it is generally very secure, but the counter must be changed at times to maintain security
